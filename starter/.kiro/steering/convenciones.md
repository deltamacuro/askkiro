# Convenciones del Proyecto

## Idioma
- Todos los mensajes de error deben estar en español.

## Nomenclatura
- Usar camelCase para variables y funciones.

## Documentación
- Todas las funciones deben tener comentarios JSDoc.

## Manejo de errores
- Siempre manejar errores con try/catch.

## Estructura de archivos
- El código debe estar separado en archivos independientes: HTML (.html), CSS (.css) y JavaScript (.js).

## Estilos visuales
- Color principal: #FF9900
- Color secundario: #000000
- Tipografía principal: 'Roboto', sans-serif (importar desde Google Fonts)
