🌡️ Kiro Workshop

Aprende a usar Steering, Skills, Agents, Hooks, Specs y Powers construyendo un conversor de temperaturas.

Sigue los pasos a tu ritmo. Clona el repo, abre Kiro, y deja que te guíe paso a paso.

Guión para dictar el workshop como clase. Cada paso incluye notas para el presentador y pausas para demo en vivo.

Usa la estructura del proyecto como base y pídele a Kiro que lo adapte a otro dominio: monedas, peso, distancias, lo que quieras.

Paso 0 de 7

0

Preparación del entorno

▸

Clona el repositorio e instala Kiro si aún no lo tienes.

git clone https://github.com/deltamacuro/kiro-paso-a-paso.git
cd kiro-paso-a-paso

Abre el proyecto en Kiro. Al abrirlo, Kiro leerá automáticamente los archivos de configuración en .kiro/.

🎤 Nota para el presentador: Muestra la estructura de carpetas .kiro/ y explica brevemente qué hay en cada subcarpeta (steering, skills, agents, hooks, specs). No entres en detalle aún.
1

Steering — Convenciones del proyecto

▸

Abre .kiro/steering/convenciones.md. Este archivo le dice a Kiro las reglas del proyecto: idioma, nomenclatura, JSDoc, try/catch, colores y tipografía.

Kiro lo lee automáticamente en cada interacción gracias a inclusion: always.

Pruébalo

Pídele a Kiro en el chat:

"Crea una función que convierta millas a kilómetros"

Observa cómo Kiro genera la función con JSDoc, try/catch, camelCase y mensajes de error en español, sin que se lo pidas explícitamente.

🎤 Nota: Haz la demo en vivo. Muestra el steering file primero, luego pide la función. Resalta que Kiro aplica TODAS las convenciones automáticamente.
🎨 Remix: Modifica el steering para tu dominio. Por ejemplo, cambia los colores, agrega convenciones específicas de tu equipo, o cambia el idioma de los mensajes.
2

Skills — Conocimiento de dominio

▸

Abre .kiro/skills/fisica-temperaturas/SKILL.md. Este skill le da a Kiro conocimiento sobre física de temperaturas: escalas, cero absoluto, fórmulas y validaciones.

Pruébalo

Pídele a Kiro:

"Crea una función que convierta de Celsius a Kelvin con validación"

Kiro debería validar contra el cero absoluto (-273.15°C) sin que se lo pidas, porque el skill le da ese conocimiento.

Ahora prueba un caso extremo:

"¿Qué pasa si intento convertir -300°C a Kelvin?"

Kiro debería explicar que está por debajo del cero absoluto.

🎤 Nota: Compara el resultado con y sin el skill. Sin él, Kiro no validaría el cero absoluto. Este es el momento de explicar que los skills inyectan conocimiento de dominio.
🎨 Remix: Crea un skill para tu dominio. Si haces un conversor de monedas, el skill podría tener tasas de cambio, reglas de redondeo, o restricciones de divisas.
3

Hooks — Automatización al guardar

▸

Abre .kiro/hooks/js-jsdoc-check.kiro.hook. Este hook se activa cada vez que guardas un archivo .js y le pide a Kiro que revise JSDoc y errores.

Pruébalo

Abre ejemplo.js y agrega esta función sin JSDoc:

function restar(a, b) { return a - b; }

Guarda el archivo. Observa cómo Kiro automáticamente te avisa que falta la documentación JSDoc.

🎤 Nota: Este es un momento "wow". Guarda el archivo y espera la reacción del público cuando Kiro responde automáticamente. Explica que los hooks son event-driven.
🎨 Remix: Crea hooks para tu flujo: lint al guardar, tests al crear archivos, validación de imports, etc.
4

Agents — El revisor de código

▸

Abre .kiro/agents/revisor.md. Este agente actúa como un senior developer que revisa calidad de código: JSDoc, try/catch y camelCase.

Pruébalo

En el chat de Kiro, invoca al agente:

@revisor revisa ejemplo.js

Si la función restar del paso anterior sigue sin JSDoc ni try/catch, el revisor la rechazará con detalles específicos.

Corrige el código según las indicaciones y vuelve a invocar al revisor hasta obtener ✅ APROBADO.

🎤 Nota: Muestra el ciclo completo: código malo → RECHAZADO → corrección → APROBADO. Explica que los agents son roles especializados reutilizables.
🎨 Remix: Crea agentes para tu equipo: un revisor de seguridad, un optimizador de performance, un traductor de i18n, etc.
5

Specs — Desarrollo guiado por requisitos

▸

Abre la carpeta .kiro/specs/conversion-history/. Contiene tres archivos que forman una spec completa:

  • requirements.md — Requisitos con user stories y criterios de aceptación
  • design.md — Diseño técnico con diagramas y modelos de datos
  • tasks.md — Plan de implementación con tareas incrementales

Pruébalo

Explora cada archivo para ver cómo Kiro estructura el desarrollo de una feature completa. El historial de conversiones que ves en la app fue construido siguiendo esta spec.

Para crear tu propia spec, pídele a Kiro:

"Crea una spec para agregar un modo oscuro al conversor"

Kiro generará requirements, design y tasks automáticamente.

🎤 Nota: Muestra los tres archivos de la spec existente. Resalta la trazabilidad: cada tarea referencia requisitos específicos. Luego haz la demo de crear una spec nueva en vivo.
🎨 Remix: Pídele a Kiro que cree una spec para una feature de tu dominio. Verás cómo adapta los requisitos y el diseño al contexto de tu proyecto.
6

Powers — Empaquetando todo

▸

Abre powers/temperature-dev/POWER.md. Un Power empaqueta steering, skills, agents y hooks en un solo paquete distribuible.

Este power incluye todo lo que viste en los pasos anteriores. Cualquier desarrollador que lo instale tiene inmediatamente:

  • ✅ Convenciones del equipo (steering)
  • ✅ Conocimiento de física (skill)
  • ✅ Revisor de calidad (agent)
  • ✅ Validación automática al guardar (hook)

Pruébalo

Imagina que un nuevo dev se une al equipo. En vez de leer un wiki de 20 páginas, instala el power y Kiro ya sabe todo lo que necesita.

🎤 Nota: Este es el cierre conceptual. Muestra cómo todo lo anterior se empaqueta. Haz la analogía: "Es como un npm package, pero para el conocimiento del equipo."
🎨 Remix: Crea tu propio power para tu equipo o proyecto. Empaqueta tus convenciones, conocimiento de dominio y automatizaciones.
