Tutorial interactivo para Kiro

De vibe coding
a software real

3 fases. 9 misiones. 30 minutos.

Cada mision desbloquea una feature de Kiro. Copia el prompt, pegalo en el chat, y mira lo que pasa.

Empezar Necesitas ayuda para configurar?
Fase 1

Construye

Crea la app, dale reglas, y planifica una feature real.

Vibe Coding

Crea la app con un solo prompt

Un prompt. Treinta segundos. App funcionando.

Kiro Chat
Crea una web app de conversor de temperaturas

Resultado: una app funcional en el browser.

Lo que dices

"Un prompt. Treinta segundos. Tenemos una app. Pero... ¿que pasa cuando otro dev toca este codigo?"

Steering

Haz que Kiro siga TUS reglas

Cada dev le pide cosas distintas. El codigo es un caos. Arreglalo.

Kiro Chat
Crea un steering file con las convenciones de nuestro proyecto: - Los mensajes de error deben estar en espanol - Usar camelCase para variables y funciones - Todas las funciones deben tener comentarios JSDoc - Siempre manejar errores con try/catch - Separa el codigo en html, css, y js - El color principal de la app es #FF9900 y #000000 - La tipografia debe ser Roboto como primera opcion

Ahora pruebalo:

Kiro Chat
Refactoriza el conversor de temperaturas siguiendo las convenciones del proyecto.

Resultado: JSDoc, try/catch, errores en espanol. Sin repetir las reglas.

Lo que dices

"No le repeti las reglas. Las leyo solas. Para siempre, para todo el equipo."

Specs

Planifica antes de codear

El cliente quiere historial. Un prompt largo sale mal. Hagamoslo bien.

En Kiro Panel lateral → Specs → "+" → Nombre: historial-conversiones
Descripcion de la spec
Agregar historial de conversiones al conversor. El usuario ve sus ultimas 10 conversiones, puede limpiar el historial y hacer clic en una entrada para reutilizarla. Los datos persisten con localStorage.

Resultado: requirements, design y tasks. Tu apruebas cada paso.

Lo que dices

"No es un prompt que espero que funcione. Es un plan que yo revise y aprobe."

Fase 2

Automatiza

Dale inteligencia al IDE. Hooks, agentes y conocimiento de dominio.

Hooks

El IDE que vigila por ti

Guardas un archivo. Hay un error. Te enteras en produccion. No mas.

En Kiro Panel lateral → Agent Hooks → "+"
Kiro Chat
Cuando guarde un archivo JavaScript, revisa si hay errores o funciones sin documentacion JSDoc y avisame en el chat.

Pruebalo: borra un parentesis en app.js, guarda con Cmd+S, mira el chat.

Resultado: Kiro detecta el error automaticamente.

Lo que dices

"No corri ningun comando. No instale ningun linter. El IDE reacciono solo."

Agents

Tu senior dev personal

La IA generica aprueba cualquier cosa. Dale criterio.

Kiro Chat
Crea un agente llamado "revisor" que actue como un senior developer estricto con la calidad. Solo aprueba codigo que tenga manejo de errores explicito y funciones documentadas con JSDoc.

Invocalo:

Kiro Chat
@revisor revisa app.js y dime si lo aprobarias para produccion.

Resultado: APROBADO o RECHAZADO con detalles.

Lo que dices

"Tiene el criterio de tu equipo. Uno para seguridad, otro para accesibilidad, otro para performance."

Skills

Dale expertise a Kiro

Steering son las reglas. Skills es el conocimiento. Se activa solo cuando lo necesita.

Kiro Chat
Crea una skill llamada "fisica-temperaturas" que sepa sobre escalas de temperatura, sus rangos fisicos validos, y casos extremos como el cero absoluto.

Pruebalo:

Kiro Chat
¿Que pasa si el usuario intenta convertir -300°C?

Resultado: Kiro sabe que -300°C esta por debajo del cero absoluto.

Lo que dices

"No cargue nada manualmente. Kiro supo cuando necesitaba ese conocimiento."

Fase 3

Conecta

Conecta Kiro al mundo real. Internet, AWS y el ecosistema.

Web Search

Busca sin salir del IDE

Necesitas un dato. Sales al browser. Rompe el flow. No mas.

Kiro Chat
¿Cual fue la temperatura mas baja registrada en la Tierra y donde?

Resultado: dato real con fuente, directo en el chat.

Lo que dices

"No abri ninguna pestana. La informacion llego donde la necesitaba."

MCP

Conecta con AWS en tiempo real

La IA generica no sabe que servicios de AWS existen hoy. Conectala.

En Kiro Abre .kiro/settings/mcp.json y agrega el servidor de documentacion de AWS.
Kiro Chat
¿Cual es la mejor forma de hacer deploy de esta app a un servicio de hosting estatico en AWS? Busca en la documentacion oficial.

Resultado: respuesta basada en docs oficiales de AWS.

Lo que dices

"Es la documentacion oficial, leida en tiempo real, aplicada a tu proyecto."

Powers

Empaqueta todo en un clic

Todo funciona pero vive disperso. Empaquetalo para tu equipo.

Un Power empaqueta MCP + steering + hooks + onboarding en una unidad instalable. Usa "Build a Power" para crear el tuyo.

Resultado: un paquete que cualquier dev instala y tiene todo.

Lo que dices

"Con MCP conectaste la IA a una herramienta. Con Powers le diste el manual. Un clic."

Empezamos con un prompt de una linea. Agregamos las reglas del equipo, una feature con plan aprobado, automatizacion que corre sola, un agente con criterio propio, expertise que se activa cuando la necesitas, busqueda en internet sin salir del IDE, conexion directa con la documentacion oficial de AWS, y un ecosistema de herramientas listo con un clic. Eso no es vibe coding. Eso es software.

Ayuda

Descargalo gratis desde kiro.dev. macOS, Windows y Linux.

Como ver la app en el browser

Instala Live Server. Click derecho en index.html → "Open with Live Server".

Como descargar el proyecto

git clone https://github.com/deltamacuro/kiro-paso-a-paso.git o descarga ZIP.
