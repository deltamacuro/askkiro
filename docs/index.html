Antes de empezar

Necesitas dos cosas: Kiro y el proyecto. Si ya las tienes, salta directo a la sesion 1.

1

Instala Kiro

Descargalo gratis desde la pagina oficial. Disponible para macOS, Windows y Linux.

Descargar Kiro →
2

Descarga el proyecto

Elige la opcion que prefieras:

Si tienes git instalado, copia y pega en tu terminal:

git clone -b modo-self-service https://github.com/deltamacuro/kiro-paso-a-paso.git

Sin terminal, sin git. Descarga el ZIP y descomprimelo donde quieras.

Descargar ZIP →

https://github.com/deltamacuro/kiro-paso-a-paso.git

File → Open Folder → selecciona la carpeta que acabas de clonar o descomprimir. Asegurate de que Kiro este en modo Autopilot (lo ves abajo a la derecha).

2

Las reglas del equipo

Steering ▸

La IA no sabe nada de tu equipo. Hasta que le dices.

El problema: cada dev le pide cosas distintas a la IA. El codigo es inconsistente.

Prompt Crea un steering file con las convenciones de nuestro proyecto: - Los mensajes de error deben estar en espanol - Usar camelCase para variables y funciones - Todas las funciones deben tener comentarios JSDoc - Siempre manejar errores con try/catch - Separa el codigo en html, css, y js - El color principal de la app es #FF9900 y #000000 - La tipografia debe ser Roboto como primera opcion

Kiro crea .kiro/steering/convenciones.md. Abrelo y muestralo. Ahora pruebalo:

Prompt Refactoriza el conversor de temperaturas siguiendo las convenciones del proyecto.

Senala en el diff: JSDoc en cada funcion, errores en espanol, try/catch donde antes no habia nada.

Nota para el presentador

"No le repeti las reglas. Las leyo solas. Esto aplica en cada prompt, para siempre, para todo el equipo."

Remix

Cambia las convenciones a las de tu equipo: otro idioma, otro estilo, otros colores.

3

Una feature de verdad, con plan

Specs ▸

El cliente quiere historial de conversiones. Hagamoslo bien.

Un prompt largo para una feature compleja suele salir mal. La IA adivina lo que quieres.

En Kiro

Panel lateral → Specs → boton "+" → Nombre: historial-conversiones

Descripcion Agregar historial de conversiones al conversor. El usuario ve sus ultimas 10 conversiones, puede limpiar el historial y hacer clic en una entrada para reutilizarla. Los datos persisten con localStorage.

Kiro genera 3 documentos: requerimientos, diseno y tareas. Aprueba cada seccion.

Nota para el presentador

"No es un prompt que espero que funcione. Es un plan que yo revise y aprobe. Kiro lo ejecuta paso a paso."

4

El IDE que trabaja solo

Hooks ▸

¿Y si el IDE pudiera vigilar el codigo por ti?

En Kiro

Panel lateral → Agent Hooks → "+"

Prompt Cuando guarde un archivo JavaScript, revisa si hay errores o funciones sin documentacion JSDoc y avisame en el chat.

Demo 1: Abre app.js, borra un parentesis, guarda con Cmd+S. Mira el chat.

Demo 2: Crea utils.js con una funcion sin documentar:

function sumar(a, b) { return a + b; }

Guarda. El hook detecta que falta JSDoc.

Nota para el presentador

"No corri ningun comando. No instale ningun linter. El IDE reacciono solo al guardar."

5

Agentes con criterio propio

Agents ▸

¿Y si pudieras tener un senior dev disponible 24/7?

La IA generica aprueba cualquier cosa. No tiene criterio de tu equipo.

Prompt Crea un agente llamado "revisor" que actue como un senior developer estricto con la calidad. Solo aprueba codigo que tenga manejo de errores explicito y funciones documentadas con JSDoc.

Luego invocalo:

Prompt @revisor revisa app.js y dime si lo aprobarias para produccion.
Nota para el presentador

"No es un asistente generico. Tiene el criterio de tu equipo. Puedes tener uno para seguridad, otro para accesibilidad, otro para performance."

6

Expertise bajo demanda

Skills ▸

Steering siempre esta activo. Skills se activan cuando las necesitas.

No quieres cargar conocimiento especializado en cada prompt. Contexto irrelevante es contexto desperdiciado.

Prompt Crea una skill llamada "fisica-temperaturas" que sepa sobre escalas de temperatura, sus rangos fisicos validos, y casos extremos como el cero absoluto.

Kiro crea .kiro/skills/fisica-temperaturas/SKILL.md. Ahora pruebalo:

Prompt ¿Que pasa si el usuario intenta convertir -300°C?

Kiro activa la skill automaticamente y responde: el cero absoluto es -273.15°C, ese valor es fisicamente imposible.

Nota para el presentador

"No cargue nada manualmente. Kiro supo cuando necesitaba ese conocimiento. Steering son las reglas. Skills es la expertise."

7

La IA que busca sola

Web Search ▸

¿Para que cambiar de pestana si Kiro puede buscar por ti?

Estas en el IDE y necesitas un dato puntual. Sales al browser, buscas, vuelves. Rompe el flow.

Prompt ¿Cual fue la temperatura mas baja registrada en la Tierra y donde?

Kiro busca en internet en tiempo real y responde con el dato y la fuente.

Prompt ¿Cual es el estandar ISO para representar temperaturas en interfaces de usuario?
Nota para el presentador

"No abri ninguna pestana. No copie nada. La informacion llego al contexto donde la necesitaba."

8

Conectado al ecosistema AWS

MCP ▸

Kiro no vive en una burbuja. Se conecta a tus herramientas.

La IA generica da respuestas genericas. No sabe que servicios de AWS existen hoy.

En Kiro

Abre .kiro/settings/mcp.json y agrega:

{ "mcpServers": { "awslabs.aws-documentation-mcp-server": { "command": "uvx", "args": ["awslabs.aws-documentation-mcp-server@latest"], "env": { "FASTMCP_LOG_LEVEL": "ERROR" }, "disabled": false, "autoApprove": [] } } }
Prompt ¿Cual es la mejor forma de hacer deploy de esta app a un servicio de hosting estatico en AWS? Busca en la documentacion oficial.

Kiro consulta la documentacion real de AWS en tiempo real.

Nota para el presentador

"No es una respuesta generica. Es la documentacion oficial de AWS, leida en tiempo real, aplicada a tu proyecto."

9

Todo empaquetado

Powers ▸

MCP es el protocolo. Powers es la experiencia lista para usar.

Todo lo que construimos funciona, pero vive disperso en .kiro/. Si otro dev empieza de cero, tiene que configurar todo a mano.

Un Power empaqueta MCP servers + steering + hooks + instrucciones de onboarding en una sola unidad instalable.

Ejemplo: Usa el Power "Build a Power" para crear tu propio Power que empaqueta todo lo que hicimos en esta demo.

Los Powers del catalogo oficial estan curados por partners (AWS, Stripe, Figma, etc.) y la comunidad.

Nota para el presentador

"Con MCP conectaste la IA a una herramienta. Con Powers le diste el manual de como usarla. Un clic. Sin configuracion. Listo para el equipo."
