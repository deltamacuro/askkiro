Paso 0 de 9

1

El punto de partida

Vibe Coding ▸

"Así empieza todo el mundo."

Abre Kiro. Modo Autopilot. Escribe esto en el chat:

Crea una web app de conversor de temperaturas

Abre el resultado en el browser. Funciona. Se ve bien.

🎤 Qué dices: "Un prompt. Treinta segundos. Tenemos una app. Pero... ¿qué pasa cuando otro dev del equipo toca este código? ¿Sabe qué convenciones usar? ¿La IA las conoce?"
2

Las reglas del equipo

Steering ▸

La IA no sabe nada de tu equipo. Hasta que le dices.

El problema: cada dev le pide cosas distintas a la IA. El código es inconsistente. Nombres en inglés, errores en inglés, sin documentación.

Escribe en el chat:

Crea un steering file con las convenciones de nuestro proyecto: - Los mensajes de error deben estar en español - Usar camelCase para variables y funciones - Todas las funciones deben tener comentarios JSDoc - Siempre manejar errores con try/catch - Separa el codigo en html, css, y js - El color principal de la app es #FF9900 y #000000 - La tipografia debe ser Roboto como primera opcion

Kiro crea .kiro/steering/convenciones.md. Ábrelo y muéstralo.

Ahora pruébalo:

Refactoriza el conversor de temperaturas siguiendo las convenciones del proyecto.

Señala en el diff: JSDoc en cada función, errores en español, try/catch donde antes no había nada.

🎤 Qué dices: "No le repetí las reglas. Las leyó solas. Esto aplica en cada prompt, para siempre, para todo el equipo."
🎨 Remix: Cambia las convenciones a las de tu equipo: otro idioma, otro estilo, otros colores.
3

Una feature de verdad, con plan

Specs ▸

El cliente quiere historial de conversiones. Hagámoslo bien.

Un prompt largo para una feature compleja suele salir mal. La IA adivina lo que quieres.

Panel lateral → Specs → botón "+" → Nombre: historial-conversiones

Agregar historial de conversiones al conversor. El usuario ve sus últimas 10 conversiones, puede limpiar el historial y hacer clic en una entrada para reutilizarla. Los datos persisten con localStorage.

Kiro genera 3 documentos:

  • Requerimientos — qué debe hacer
  • Diseño — cómo lo va a implementar
  • Tareas — pasos concretos de código

Aprueba cada sección. Que se vea que tú decides.

🎤 Qué dices: "No es un prompt que espero que funcione. Es un plan que yo revisé y aprobé. Kiro lo ejecuta paso a paso."
4

El IDE que trabaja solo

Hooks ▸

¿Y si el IDE pudiera vigilar el código por ti?

Panel lateral → Agent Hooks → "+" → escribe:

Cuando guarde un archivo JavaScript, revisa si hay errores o funciones sin documentación JSDoc y avísame en el chat.

Demo 1: Abre app.js, borra un paréntesis, guarda con Cmd+S. Mira el chat.

Demo 2: Crea utils.js con una función sin documentar:

function sumar(a, b) { return a + b; }

Guarda. El hook detecta que falta JSDoc.

🎤 Qué dices: "No corrí ningún comando. No instalé ningún linter. El IDE reaccionó solo al guardar. Un solo hook. Detecta errores y documenta funciones."
5

Agentes con criterio propio

Agents ▸

¿Y si pudieras tener un senior dev disponible 24/7?

La IA genérica aprueba cualquier cosa. No tiene criterio de tu equipo.

Crea un agente llamado "revisor" que actúe como un senior developer estricto con la calidad. Solo aprueba código que tenga manejo de errores explícito y funciones documentadas con JSDoc.

Luego invócalo:

@revisor revisa app.js y dime si lo aprobarías para producción.
🎤 Qué dices: "No es un asistente genérico. Tiene el criterio de tu equipo. Puedes tener uno para seguridad, otro para accesibilidad, otro para performance."
6

Expertise bajo demanda

Skills ▸

Steering siempre está activo. Skills se activan cuando las necesitas.

No quieres cargar conocimiento especializado en cada prompt. Contexto irrelevante es contexto desperdiciado.

Crea una skill llamada "fisica-temperaturas" que sepa sobre escalas de temperatura, sus rangos físicos válidos, y casos extremos como el cero absoluto.

Kiro crea .kiro/skills/fisica-temperaturas/SKILL.md. Ahora pruébalo:

¿Qué pasa si el usuario intenta convertir -300°C?

Kiro activa la skill automáticamente y responde: el cero absoluto es -273.15°C, ese valor es físicamente imposible.

🎤 Qué dices: "No cargué nada manualmente. Kiro supo cuándo necesitaba ese conocimiento. Steering son las reglas. Skills es la expertise. Cada una carga cuando corresponde."
7

La IA que busca sola

Web Search ▸

¿Para qué cambiar de pestaña si Kiro puede buscar por ti?

Estás en el IDE y necesitas un dato puntual. Salís al browser, buscás, volvés. Rompe el flow.

¿Cuál fue la temperatura más baja registrada en la Tierra y dónde?

Kiro busca en internet en tiempo real y responde con el dato y la fuente.

Segundo ejemplo:

¿Cuál es el estándar ISO para representar temperaturas en interfaces de usuario?
🎤 Qué dices: "No abrí ninguna pestaña. No copié nada. La información llegó al contexto donde la necesitaba."
8

Conectado al ecosistema AWS

MCP ▸

Kiro no vive en una burbuja. Se conecta a tus herramientas.

La IA genérica da respuestas genéricas. No sabe qué servicios de AWS existen hoy.

Abre .kiro/settings/mcp.json y agrega:

{ "mcpServers": { "awslabs.aws-documentation-mcp-server": { "command": "uvx", "args": ["awslabs.aws-documentation-mcp-server@latest"], "env": { "FASTMCP_LOG_LEVEL": "ERROR" }, "disabled": false, "autoApprove": [] } } }

Ahora pregunta:

¿Cuál es la mejor forma de hacer deploy de esta app a un servicio de hosting estático en AWS? Busca en la documentación oficial.

Kiro consulta la documentación real de AWS en tiempo real.

🎤 Qué dices: "No es una respuesta genérica. Es la documentación oficial de AWS, leída en tiempo real, aplicada a tu proyecto. Eso es MCP: conecta la IA con tus herramientas reales."
9

Todo empaquetado: Powers

Powers ▸

MCP es el protocolo. Powers es la experiencia lista para usar.

Todo lo que construimos funciona, pero vive disperso en .kiro/. Si otro dev empieza de cero, tiene que configurar todo a mano.

Un Power empaqueta MCP servers + steering + hooks + instrucciones de onboarding en una sola unidad instalable.

Ejemplo: Usa el Power "Build a Power" para crear tu propio Power que empaqueta todo lo que hicimos en esta demo.

Los Powers del catálogo oficial están curados por partners (AWS, Stripe, Figma, etc.) y la comunidad.

🎤 Qué dices: "Con MCP conectaste la IA a una herramienta. Con Powers le diste el manual de cómo usarla. Un clic. Sin configuración. Listo para el equipo."
